MyLOL英雄联盟查询小助手

1. MyLOL英雄联盟查询小助手

1.1. 有话说

最近都很忙啊啊，忙着找实习，忙着学习，忙着公众号的事情，忙着软件杯的事情，忙得不可开交…

而且最近也在参加Google Study Jams活动，这个活动是Android面向初学者的，虽然说自己也算是Android刚刚入了门，但官方的活动还是想去参加一下，向Google的工程师学习一下还是很有帮助的！！

这次活动也接近尾声了，最后要求每一位学员都提交一个结业的作业，就是要自己动手写一个Android App出来。 我选择了以英雄联盟为主题去制作一个小app，但是上网找数据接口的时候发现，官方并没有提供英雄联盟中国区的玩家数据接口！！

这真是bi le gou 了，不过还是有一些第三方的api接口，不过查询到的数据也是太抠脚了，更新有点慢…凑合着用把….

这个app当然是要采用MD设计的，个人是非常喜欢这个设计啊啊，喜欢得不得了，但是还没时间去深入研究Material Design，因为最近都要补基础 (基础较弱的孩子伤不起)

下面贴一下采用的一些库：

compile 'com.android.support:appcompat-v7:23.3.0'
compile 'com.android.support:design:23.3.0'
compile 'com.kymjs.rxvolley:rxvolley:1.0.7'
compile 'com.kymjs.rxvolley:bitmapcore:1.0.7'
compile 'com.android.support:cardview-v7:23.1.1'
compile 'com.google.code.gson:gson:2.2.4'

主要用到了com.android.support:design:23.3.0库里面的一些控件，实现了一些比较不错的效果

网络使用到了RxVolley ，感觉使用起来很方便,可以适当地对他进行封装，满足自己项目的需求就好；

概述 RxVolley是一个基于Volley的网络请求库； 同时支持RxJava； 可以选择使用OKHttp替代默认的 HttpUrlConnection 做网络请求； 可以选择使用图片加载功能(复用的网络请求将有效减少apk体积)； 移除了原Volley的 HttpClient 相关 API ，可在 API23 环境编译； 内置了RxBus的实现，可有效替换掉EventBus等相关库；

这是RxVolley的Gihub项目：RxVolley

整个项目都没有用到一些架构的东西，代码也写得很乱，Fragment切换的时候还有些卡顿现象，这些在后面再解决吧
