EventBus使用指南

前天刚完成了公司项目的一期版本研发，现在的空闲时间比较多，于是就开始学习使用新的开源库，在做项目过程中也看到项目中集成了EventBus,之前也没怎么看过这个开源库…因此现在也想学一下它的使用方法，后面还想深入到源码来分析一下它的原理。鉴于自己还是个菜鸟，有什么错漏或者不对的地方，还请各位大神提出。

首先说明一下EventBus是个什么东西，EventBus是一个事件总线，总线我们都知道是计算机各种功能部件之间传送信息的公共通信干线。事件总线，顾名思义就是各个部件之间传事件的公共通信干线咯。它用于简化Android程序中各个组件，线程之间的事件传递。它使用观察者模式，也就是订阅/发布的模式，来达到解耦的目的。分别有publisher和subscribe两个角色，第一个负责发布事件，第二个负责订阅接收事件并处理。

EventBus官网的图片

使用场景

很多时候，我们需要在Activity内部，Activity和Activity间，Fragment和Fragment间，或者其他组件之间传递数据，数据不限于简单的基本数据，也有可能是复杂的对象数据，那么我们一般都会使用回调接口、Handler、广播等方法去传递我们的事件和数据。这样做的后果是使得我们的代码变得很复杂，很乱，没有统一的事件管理。因此，在Android程序中，对应以上的一些情景，我们可以选择使用EventBus来进行统一的事件管理。

使用优点

  • 简化组件之间的通信
  • 使得事件的发送者和接收者得以解耦
  • 可以在Avtivity间、fragment间或者后台线程中很好的使用
  • 避免了复杂且容易出错的依赖关系，以及生命周期问题
  • 速度快
  • 轻量级，jar包<50k
  • 超过10000+APP集成了EventBus,说明它的受欢迎程度高，可靠
  • 拥有其他一些高级功能，如线程间通信，订阅者优先级设置等

使用方法(按照EventBus官网说明的使用步骤)

1.在项目中导入组件

打开App的build.gradle,在 dependencies中添加依赖

2.定义事件

所有能被实例化为 Object 的实例都可以作为事件

3.定义订阅者方法

在订阅方法上添加@Subcribe注解，其中注解有三个参数，分别是threadMode,priority,sticky.
其中threadMode为回调所在线程(具体后面说明)，priority为优先级，sticky为是否接收粘性事件。
方法名没有特定要求，但是只能与一个参数，而且注册了监听的模块必须有一个标注了 Subscribe 注解方法，不然在 register 时会抛出异常：
Subscriber class XXX and its super classes have no public methods with the @Subscribe annotation

说明：

threadMode , 用于指定此函数运行的线程, 是一个Enum, 有4个常量, MAIN BACKGROUND ASYNC POSTING, 默认为POSTING

  • ThreadMode.MAIN 在主线程中运行

  • ThreadMode.POSTING 跟消息发送者在同一线程运行

  • ThreadMode.BACKGROUND 后台线程, 如果发送消息的线程就是后台线程,就直接执行; 如果不是, 则会把消息放在队列中,依次执行

  • ThreadMode.ASYNC 后台线程, 消息会在单独的线程中执行,用了线程池,多个消息会同时执行

priority 优先级, 值越小优先级越低,当有多个方法处理同一个消息时,处理的顺序,默认为0

sticky 是否接收黏性消息, 和黏性广播相同, 默认为false

4.注册EventBus/取消注册EventBus

在需要用到的组件中，如Activity,我们一般在组件初始化时注册，在组件销毁时取消注册

5.发送事件

可以调用 post 或者 postSticky
